•　使用 ===== 表示高阶标题，使用 ——— 表示次阶标题。

1.2　引用

1.2.1　说明

使用 > 表示引用， >> 表示引用里面再套一层引用，依次类推。当 > 与后面文字间添加5个blank时，块注释的文字会有变化。

1.2.3　注意

1.　如果 > 和 >> 嵌套使用的话，从 >> 退到 > 时，两行之间必须要加一个回车或者 > 作为过渡，否则默认为下一行和上一行是同一级别的引用。
2.　引用标记里可以使用其他标记，如：有序列表或无序列表标记，代码标记等。

1.9　链接

1.9.1　说明

Markdown 支持两种形式的链接语法： 行内式和参考式两种形式。

行内式链接使用[](link "Optional title") 表示。其中：
•　[] 内的内容为要添加链接的文字。
•　link 为链接地址。
•　Optional title 为显示标题。显示效果为在你将鼠标放到链接上后，会显示一个小框提示，提示的内容就是 Optional title 里的内容。

参考式链接则使用以下方式，
[Baidu][1]
[1]:www.baidu.com "百度一下，你就知道"，其中：
•　第一个方括号内的内容为要添加链接的文字。
•　第二个方括号里的内容为用以辨识链接的标记。
•　第三个方括号里的内容与第二个对应。
•　冒号后面为链接地址。
•　最后为显示标题。

自动链接：Markdown 支持以比较简短的自动链接形式来处理网址和电子邮件信箱，只要是用<>包起来， Markdown 就会自动把它转成链接。一般网址的链接文字就和链接地址一样。

1.9.2　示例

例1：行内式链接

以上内容显示如下（链接可点击）：

Baidu

例2：参考式链接

以上内容显示如下（链接可点击）：

Baidu

例3：自动链接

以上内容显示如下：

https://www.baidu.com/

1.9.3　注意

•　关于链接使用的更多技巧和限制请参考：Markdown 语法说明

2.2　脚注

2.2.1　说明

在需要添加注脚的文字后加上脚注名字[^注脚名字],称为加注。 然后在文本的任意位置(一般在最后)添加脚注，脚注前必须有对应的脚注名字。

注意：经测试注脚与注脚之间必须空一行，不然会失效。成功后会发现，即使你没有把注脚写在文末，经Markdown转换后，也会自动归类到文章的最后。

2.2.2　示例

2.2.3　注意

•　脚注自动被搬运到最后面，请到文章末尾查看，并且脚注后方的链接可以直接跳转回到加注的地方。
•　脚注语法支持与否与所用的编辑器有关，我所用的Atom不支持脚注。
